# ai-scene-to-video

> **从视频素材到成品短视频的自动化内容生产流水线**
> *Scene → Insight → Script → Clip → Publish*

---

## 一、项目背景

在房产短视频内容生产中，存在以下现实问题：

* 中介拍摄了大量**原始房源视频素材**，但：

  * 不会剪辑
  * 不会写解说文案
  * 内容质量不稳定
* 人工剪辑 + 配音成本高、效率低，**无法规模化**

与此同时：

* 大模型已经具备**视频理解、文本生成能力**
* 但 **“直接让 AI 剪视频”不可控、不稳定**

因此，本项目的目标不是“炫技式 AI”，而是：

> **用可控的 AI + 规则化工程，把房源视频批量变成“能发平台的看房短视频”**

---

## 二、项目目标（明确约束）

### ✅ 做什么

* 输入：

  * 房产中介拍摄的 **原始竖屏视频素材**
* 输出：

  * 房源视频的**结构化理解结果**
  * 基于理解结果生成的**解说脚本**
  * 自动剪辑、配音、字幕合成的**成品短视频**

### ❌ 不做什么

* 不判断房源投资价值
* 不评价学区、地段、价格合理性
* 不生成不可验证或夸大的宣传内容

---

## 三、核心设计原则

### 1️⃣ 结构化优先，而不是自由生成

* 所有 AI 输出 **必须是结构化 JSON**
* 自然语言只出现在最终对外文案阶段

### 2️⃣ AI 决策，程序执行

* AI 负责：

  * 视频语义理解
  * 卖点抽象
  * 剪辑方案生成
* 程序负责：

  * ffmpeg 裁剪
  * 转场、字幕、配音合成

### 3️⃣ 可验证、可约束

* 只允许基于视频**明确可见信息**进行分析
* 推断类结论必须有规则来源

---

## 四、整体架构

```text
原始视频
   ↓
关键帧 / 音频抽取
   ↓
视频理解（Vision LLM）
   ↓
卖点与结构抽象（规则层）
   ↓
解说脚本生成（LLM）
   ↓
剪辑方案生成（JSON）
   ↓
ffmpeg / TTS / 字幕合成
   ↓
成品短视频
```

## 五、模块说明

### 1️⃣ vision/ —— 视频理解模块

**职责**

* 对房源视频进行语义理解
* 输出可验证的结构化信息

**典型输出**

```json
{
  "scenes": [
    {
      "time": "00:03-00:10",
      "room": "客厅",
      "features": ["大窗", "采光明显"]
    }
  ]
}
```

**可选模型**

* GPT-4o
* Qwen2-VL
* Gemini 1.5（长视频）

---

### 2️⃣ analysis/ —— 卖点与内容抽象层

**职责**

* 将“视频可见特征”映射为“可对外表达的卖点”
* 引入房产行业表达规则

**示例规则**

| 视频特征 | 可表达卖点 |
| ---- | ----- |
| 大窗   | 采光表现好 |
| 空间开阔 | 动线流畅  |

---

### 3️⃣ script/ —— 解说脚本生成

**职责**

* 根据卖点与视频节奏生成解说文本

**特点**

* 固定结构（钩子 → 分段 → 总结）
* 多模板随机，避免 AI 痕迹

---

### 4️⃣ edit/ —— 自动剪辑执行

**职责**

* 根据剪辑方案 JSON 执行视频裁剪
* 合成字幕、配音、背景音乐

**技术栈**

* ffmpeg
* MoviePy（可选）

---

### 5️⃣ pipeline/ —— 流水线编排

* 串联各模块
* 支持批量处理
* 预留 n8n / Cron / API 接口
